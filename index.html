GIT NOTES

1. What is GIT ?

Git ia s distributed version control system that tracks versions of files. It is often used to control source code by programmers

2. What is Github ?

Github is a developer storage platform that allows developers to create, store, manage and share their code and allows developers to work together. It uses Git Software, which provides distributed version control of access control, bug tracking, software feature requests, task managemnet, continous integration for every project

3. What is git init ?

The git init command creates a new Git repository.It can be used to convert an existing, unversioned project to a Git repository or initialize a new, empty repository. Most other Git commands are not available outside of an initialized repository, so this is usually the first command you'll run in a new project
Or
Initializes a new Git repository in the current directory. This creates a hidden .git folder, which tracks changes in the project.

4. What is git status ?

The git status command displays the state of the working directory and the staging area, It lets you see which changes have been staged, which haven't, and which files aren't being tracked by Git.

5. What is git add ?

The git add command adds a change in the working directory to the staging area, It tells Git that you want to include updates to a particular file in the next commit.

6. What is the use of git commit ?

The "commit" command is used to save your changes to the local repository
Or
Commits the changes in the staging area to the repository with a message "first commit". This snapshot of the project can be referenced later.
git commit -m "first commit"

7. What is git restore ?

git restore is used to get back the files from staging area
git restore --staged filename
also used to restore uncommited chnages
git restore filename

8. What is git log ?

It displays the commit history,
git log
git log --oneline (breif history)

9. What is git branch -M main ?

Renames the default branch to main. -M is a forced rename flag, often used to switch the default branch from master to main.

10. What is git remote add origin https://github.com/usename/repository.git ?

Sets up a remote repository named origin with the specified URL. This allows you to push and pull changes from GitHub.
git remote add origin https://github.com/Sujan2332/GIT.git

Or

The git remote add command will create a new connection record to a remote repository.
git remote add name url

11. What is git push -u origin main ?

Pushes the local main branch to the origin remote repository on GitHub. The -u flag sets up origin/main as the default upstream branch, so future pushes can be done with just git push.

12. What is git config ?

git config is a command in Git used to set configuration options that control Git's behavior, including user information, repository settings, and preferences.
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

13. What is git push --set-upstream GIT master ?

The command git push --set-upstream GIT master is used to push a local branch to a remote repository and set it up so that future pushes to this branch are associated with the specified remote repository and branch.

14. What is git push ?

The git push command is used to uplaod local repository content to a remote/global repository

# GIT LIVE WEBSITE HOSTING

Option 1: (for basic html and css files)

git add .
git commit -m "Add website content"
git push origin main

  • Go to your repository on GitHub.
  • Go to the "Settings" tab.
  • Scroll down to "Pages" in the sidebar on the left.
  • Under "Source", select the branch you want to publish. For example, you can select main or master as the source, and specify the root directory or /docs folder as the location.
  • Click "Save". After a few moments, your website will be live at https://username.github.io/repository-name

Option 2: (gh-pages (for react and node files))

  • Install the gh-pages package (run this command in the root directory of your project):

    npm install --save-dev gh-pages
  • Configure your package.json file:

    Add a homepage field with the URL of your GitHub Pages site:
    "homepage": "https://username.github.io/repository-name"
  • Add deploy scripts under "scripts":

    "scripts": {
    "predeploy": "npm run build", // Builds the project
    "deploy": "gh-pages -d build" // Deploys the build folder
    }
  • Deploy your site:

    npm run deploy
    This command will build your project and publish the build directory to a gh-pages branch.
  • Access your website at https://username.github.io/repository-name
